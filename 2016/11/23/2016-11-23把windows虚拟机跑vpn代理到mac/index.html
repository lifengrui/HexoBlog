把windows虚拟机跑vpn代理到mac

发表于 2016-11-23   |     |   阅读次数

mac下使用vpn遇到的问题

我用的是绿豆vpn，好久之前买的，记得当时300块买了2年送300天，而且当时有浏览器插件，很方便，所以也没怎么犹豫。

目前我在使用时遇到一个困境：window下使用没有问题，我的ios手机连接不上，但不怎么使用，所以就当没有问题，但是呢，现在在mac下使用unity，下载资源需要翻墙，但是mac下不提供vpn，绿豆vpn连最基本的服务器地址都不提供，所以不能是用pptp，IKEv2等，简直没人性。（我没有知识基础的猜测，要么是防止国家查封服务器，要么防止别人黑他的服务器。有清楚的可以留言说下原因。

好了，闲话就到这里。开始说说我是怎么解决的.

解决思路

1.在windows虚拟机中装vpn软件，连上vpn

2.将windows中全局的网再反向连接到mac上，这样mac就成功全局翻墙。

百度一波发现一个好的教程，教的是给学校的网络登陆客户端，把虚拟机中的网分享到mac的方法。真的很赞，我要是早些发现说不定就不用联通路由器激活插件了。单独给自己整个电信网。。。我又说废话了。。。

首先在MacVM虚拟机设置中添加网卡，一共两个

我开始失败是由于没有在VM的偏好设置里添加新的连接方式：如下图

教程中说的独占模式其实是N多虚拟机共享网络，与主机是不通的。新设置后的选项是通着，就可以使用代理了。

具体流程：

1. 安装xp

2. 设置两个连接方式

一个桥接当前获取网络的驱动，一个是自己建立的连接方式。都DHCP模式

3. 启动xp，登上vpn

4. 启用代理CCProxy

在设置->高级->网络->取消禁止外网并绑定服务器ip为自己建立的连接方式的IP！注意：不是192开头这种！如果是192，就是步骤2出现问题。（T T我在这卡了一会。。）百度云下载CCProxy

5. 在mac的网络->高级->代理：

按照下图

设置。enjoy it！

6. 最后，如果不需翻墙（没开VM），记得取消5中的对勾。

（教程适用于任何需要将虚拟机中的网代理到主机的诉求）
